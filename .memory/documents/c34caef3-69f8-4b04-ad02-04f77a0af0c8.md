# Requirements: Fuzzy Search for Research and Plans Queries

## User request

Add support for fuzzy search for text queries over research and plan memories.

## Clarified requirements

- Fuzzy matching enabled by default for query-research and query-plans.
- Direct/exact matches must continue to work without regression.
- Fuzzy matching compares against both document summary and body text.
- Output uses a combined ranked list rather than separate direct/fuzzy sections.
- A dedicated fuzzy matching crate may be added if appropriate.

## Inferred requirements

- Preserve type filtering: query-research returns only RESEARCH, query-plans returns only PLAN.
- Existing behavior where research/plan query output does not print full document bodies must remain.
- Results must be deterministic and useful for agent consumption.
- Direct/high-confidence matches must rank above weaker fuzzy matches.
- CLI remains simple; no new required arguments.
- Tests cover typo/fuzzy cases, direct-match ordering, type filtering, and body-based fuzzy matching.

## Non-goals

- Do not change query-files behavior unless shared abstractions require harmless refactoring.
- Do not implement semantic/vector search.
- Do not change storage schema. Dependency-level in-memory ranking approach is preferred.
