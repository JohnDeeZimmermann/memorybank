# Plan: Graph-Based Weighting for Primary and Related Search Results

## Objective

Implement graph-aware ranking for Memory Bank queries so primary matches and secondary related suggestions are sorted by a blend of: direct query relevance, query-local graph relatedness, global graph authority from direct/transitive references, and recency.

## Recommended algorithm

Use two PageRank-style signals:

### 1. Global authority score
Weighted PageRank over all documents including invalidated. Nodes: all documents. Directed edges: from_document_id -> to_document_id (weight 1.0). Shared-file co-reference edges: bidirectional (weight 0.25-0.4 after fanout normalization). Damping: 0.85.

### 2. Query-local relatedness score
Personalized PageRank / random walk with restart. Nodes include invalidated contributors. Personalization vector: normalized fuzzy/text scores for text queries, normalized file-match count for file queries, mass 1.0 for the read document. Sink nodes redistribute to personalization vector.

### Creation date signal
Recency = exp(-ln(2) * age_days / 365). Parsing failures get neutral/low value.

## Score blending
Normalize each signal to [0, 1]. direct_text_final = 0.70*text_score + 0.15*personalized_pagerank + 0.10*global_authority + 0.05*recency. direct_file_final = 0.55*file_match_score + 0.25*personalized_pagerank + 0.15*global_authority + 0.05*recency. related_final = 0.60*personalized_pagerank + 0.25*global_authority + 0.15*recency.

Sort by final score desc, then exactness tier, then created_at DESC, then id ASC.

## Relevant files
src/scorer.rs, src/commands/query_text.rs, src/commands/query_files.rs, src/commands/read.rs, src/db.rs, src/store.rs, src/sql_log.rs, src/config.rs.

## Database work
Add index: CREATE INDEX IF NOT EXISTS idx_document_links_from ON document_links(from_document_id). Add bulk DB helpers: graph_documents, graph_document_links, graph_file_memberships, summaries_by_ids_bulk, related_files_for_ids_bulk.

## Graph-ranking module
Create src/graph_ranker.rs. Use dense integer-indexed adjacency, Vec<String> of IDs sorted by ID, HashMap<String, usize> for ID lookup. Pure Rust power iteration with f64. No external graph dependencies.

File co-reference: collect member document indexes per file, skip files with fanout above max_file_fanout (default 100), bidirectional edges with weight divided by (members.len() - 1).

## Command integration
query_text.rs: load candidates, compute text scores, build GraphIndex, compute personalized + authority + recency, re-sort using direct_text_final, build related suggestions from non-direct documents above threshold, sort by related_final. query_files.rs: compute file-match scores, seed personalized PageRank, sort direct with direct_file_final, rank related with related_final. read.rs: seed with the read document, rank related with related_final.

## Configuration
Add GraphRankingConfig with serde defaults: enabled=true, max_related_suggestions=20, max_file_fanout=100, max_iterations=80, tolerance=1e-6, damping=0.85, recency_half_life_days=365.0.

## Performance safeguards
Bulk-load once per command, use dense adjacency vectors, normalize outgoing edge weights during construction, cap iterations with L1 convergence, cap file fanout, cap related suggestions, fall back to current ordering if no edges/seeds.
